ArcPay is a control plane and developer distribution layer for agent payments, treasury policies, paid APIs, and verifiable execution.

This demo shows ArcPay Somnia, a live wallet-first operating account for AI-agent businesses on Somnia Testnet.

The problem is that autonomous agents can discover tools and pay for work, but operators still need budget controls, receipts, privacy boundaries, and audit evidence before trusting them with treasury access.

ArcPay solves that by routing agent work through one policy layer: x402 payment requirements, on-chain orders, treasury limits, privacy intents, risk records, reputation, and audit trails.

First, the operator signs in and opens a workspace. This workspace is account-backed, so the operator can return later from another device and keep the same treasury context.

The dashboard gives the operator one console for agents, orders, payments, invoices, cards, privacy, analytics, and status.

On the Agents page, ArcPay supports bring-your-own-agent onboarding. A builder can register an agent slug, service endpoint, capabilities, and price, then expose that agent as a paid resource.

ArcPay also integrates official Somnia Agents. The app includes the Somnia platform contract, AgentRegistry, LLM Parse Website, LLM Inference, and JSON API Request IDs. Their receipts can be attached to ArcPay records before an operator marks work complete.

Now the x402 flow. A client requests a protected agent resource over HTTP. ArcPay returns a 402 payment requirement with the exact network, asset, recipient, and order details.

After payment and verification, the protected result can be unlocked. This makes ArcPay useful for paid APIs, paid MCP tools, research endpoints, and agent-to-agent services.

Policies keep that payment flow safe. Operators define budget limits, approval thresholds, allowed agents, and evidence requirements before funds move.

Privacy intents add a second boundary. Teams can commit to a payment or memo without exposing every detail immediately, then release selective evidence later for audit or settlement.

ArcPay also maps Somnia ecosystem routes into policy-gated intent flows, including dreamDEX, Somnia Exchange, Somnex, Potion Swap, and custom DEX adapters.

The important rule is that ArcPay does not treat a quote as completion. A route needs venue evidence, a Somnia transaction hash, or an ArcPay audit record before it is considered done.

The Audit and Status pages show why this matters. ArcPay keeps visible evidence across wallet actions, x402 orders, invoices, cards, privacy intents, agent receipts, and API health.

ArcPay is not only an app. It publishes a CLI, MCP server, x402 starter kit, OpenAPI endpoint, llms.txt, MCP server card, agent skills metadata, and an API catalog.

That gives Somnia builders multiple ways to integrate: human operators can use the dashboard, AI agents can call MCP tools, and developers can use the starter kit to sell or consume paid agent work.

ArcPay turns agent payments from isolated demos into a usable treasury operating system for real agent businesses on Somnia.
