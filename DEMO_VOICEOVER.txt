[0:00-0:12 Landing]
ArcPay Somnia is a control plane and developer distribution layer for agent payments, treasury policy, paid APIs, and verifiable execution on Somnia Testnet.

[0:12-0:25 Auth]
The operator starts with wallet-first access. A signed wallet session opens a persistent workspace, so the same treasury context can be used across devices.

[0:25-0:45 Dashboard]
This dashboard is the operating account for an AI-agent business: agents, orders, payments, invoices, cards, privacy, risk, status, and audit all live in one console.

[0:45-1:10 Agents]
ArcPay supports bring-your-own-agent onboarding. A builder can register an agent slug, service endpoint, capabilities, and price, then expose that agent as a paid resource.

[1:10-1:35 Operator]
Operators can issue claim codes for external agents and control webhook circuit breakers. That lets an agent join ArcPay without a manual dashboard setup, while unsafe endpoints can be paused before they waste budget.

[1:35-2:20 x402]
The x402 flow turns an agent endpoint into a paid HTTP resource. A client requests work, ArcPay returns a 402 payment requirement, the user or agent creates an on-chain order, then the result unlocks only after verification.

[2:20-2:45 Policies]
Policy is the safety layer. ArcPay checks wallet access, spend limits, allowed tokens, allowed networks, risk floors, and approval thresholds before money-moving actions proceed.

[2:45-3:10 SOMUSD cards]
SOMUSD cards give agents bounded budgets. The operator creates a card, tops it up, freezes or activates it, and the assigned agent can only spend within the card policy.

[3:10-3:35 Privacy]
Privacy intents add a second boundary. Teams can create commitment-based payments or disclosure records, then release selective evidence later for audit or settlement.

[3:35-3:55 DeFi adapters]
ArcPay also maps Somnia ecosystem routes into policy-gated intent flows, including dreamDEX, Somnia Exchange, Somnex, Potion, and custom DEX adapters. A quote is not treated as completion; ArcPay requires transaction or audit evidence.

[3:55-4:20 Audit and status]
The audit and status pages show why this matters. ArcPay keeps visible evidence across wallet actions, x402 orders, invoices, cards, privacy intents, agent receipts, API health, and contract status.

[4:20-4:45 Developer layer]
ArcPay is not only a frontend. It publishes a CLI, MCP server, x402 starter kit, OpenAPI, llms.txt, MCP server card, and agent metadata so other Somnia builders and AI agents can integrate directly.

[4:45-4:55 Close]
ArcPay turns agent payments from isolated demos into a usable treasury operating system for real agent businesses on Somnia.
