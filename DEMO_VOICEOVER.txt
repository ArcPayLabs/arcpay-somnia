[0:00-0:12 Landing]
ArcPay Somnia is a control plane and developer distribution layer for agent payments, treasury policies, paid APIs, and verifiable execution on Somnia Testnet.

[0:12-0:25 Wallet sign-in]
The operator starts by choosing an EVM wallet. ArcPay asks the wallet to add or switch to Somnia Testnet, then signs one login challenge to open the workspace.

[0:25-0:40 Dashboard]
This is the operating account for an AI-agent business. Agents, orders, cards, payments, privacy, risk, status, and audit records live in one console instead of separate scripts.

[0:40-1:05 Bring your own agent]
Here I onboard a service agent. The operator sets the agent slug, endpoint, capabilities, and price. The same agent can later be called through dashboard flows, CLI, MCP, or x402.

[1:05-1:25 Operator onboarding]
The operator can also issue claim codes for outside agents. A builder can redeem the code programmatically, while webhook circuit breakers protect the treasury from failing or unsafe endpoints.

[pause while claim fields are shown]

[1:25-1:55 x402 paid work]
ArcPay exposes paid agent work over HTTP 402. A client requests the resource, receives a payment requirement, creates or verifies an on-chain order, and only unlocks the result after evidence is available.

[pause while quote and access checks load]

[1:55-2:20 Policy enforcement]
Policy sits before wallet signing. ArcPay checks network, spend limits, allowed assets, approval thresholds, risk rules, and contractor allowlists before a payment or treasury action can move forward.

[2:20-2:50 SOMUSD agent cards]
For bounded agent budgets, ArcPay issues SOMUSD spend cards. The operator creates the card, sets the agent wallet, defines a limit, tops it up, freezes it, or reactivates it without giving unlimited treasury access.

[pause while card form is shown]

[2:50-3:15 Privacy intents]
Privacy intents create commitment-based payment or disclosure records. Teams can keep sensitive memos encrypted, then release selective evidence later for audit or settlement.

[3:15-3:40 Live DeFi execution]
ArcPay includes a live Somnia testnet swap router and STT yield vault. Swaps and yield actions open the wallet, submit real transactions, and only count as complete after tx hash proof.

[3:40-4:05 Audit and status]
The audit and status pages show the proof layer: wallet actions, x402 orders, cards, privacy intents, agent receipts, API health, contract status, and explorer links all stay reviewable.

[4:05-4:30 Wallet proof]
The wallet console shows the connected signer, Somnia Testnet, STT balance, SOMUSD balance, and direct explorer links. This is where operators verify that signed actions happened on-chain.

[4:30-4:50 Developer layer]
ArcPay is not only a frontend. It ships a CLI, MCP server, x402 starter kit, OpenAPI, llms.txt, and agent metadata, so other Somnia builders and AI agents can integrate directly.

[4:50-5:00 Close]
ArcPay turns agent payments from isolated demos into a usable treasury operating system for real agent businesses on Somnia.
